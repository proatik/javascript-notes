## 17. Miscellaneous

### ✅ `eval` & `with`

#### `eval`

Executes a **string of JavaScript code** at runtime.

```js
eval("console.log('hello')"); // hello
eval("2 + 2");                // 4

const x = 10;
eval("console.log(x)");       // 10 — has access to surrounding scope
```

#### Why to avoid `eval`

- **Security risk** — executes arbitrary code; dangerous with user input (code injection).
- **Performance** — V8 cannot optimise code inside `eval` (scope is unpredictable).
- **Debugging** — errors inside `eval` strings are hard to trace.

```js
// Never do this
const userInput = "alert('hacked')";
eval(userInput); // executes malicious code
```

**Rule:** Never use `eval`. Use `JSON.parse` for data, `Function` constructor only if absolutely necessary.

#### `with`

Extends the scope chain — adds an object's properties as local variables inside the block.

```js
const obj = { x: 10, y: 20 };

with (obj) {
  console.log(x + y); // 30 — x and y come from obj
}
```

#### Why to avoid `with`

- **Ambiguity** — impossible to know at compile time whether `x` refers to the object or an outer variable.
- **Performance** — prevents V8 from optimising the scope lookup.
- **Strict mode** — `with` throws a `SyntaxError` in strict mode.
- **Deprecated** — removed from ES5 strict mode; avoid entirely.


### ✅ Unicode & String Encoding

#### Unicode basics

Every character is assigned a **code point** — a number in the Unicode standard.

```js
"A".codePointAt(0)     // 65
"😀".codePointAt(0)    // 128512

String.fromCodePoint(65)      // "A"
String.fromCodePoint(128512)  // "😀"
```

#### UTF-16 — how JS stores strings

JavaScript strings are encoded as **UTF-16**. Most characters fit in one 16-bit **code unit**, but characters above `U+FFFF` (like most emoji) need **two code units** — a surrogate pair.

```js
"A".length        // 1
"😀".length       // 2 — surrogate pair, counts as 2 code units
[..."😀"].length  // 1 — spread uses code points, not code units
```

#### Iterating correctly over strings with emoji

```js
const str = "Hi 😀";

// Wrong — iterates over code units
for (let i = 0; i < str.length; i++) {
  console.log(str[i]); // "H", "i", " ", "?", "?" — splits emoji
}

// Correct — iterates over code points
for (const char of str) {
  console.log(char); // "H", "i", " ", "😀"
}

[...str] // ["H", "i", " ", "😀"]
```

#### Unicode escapes

```js
"\u0041"  // "A"  — 4-hex escape (BMP only, U+0000 to U+FFFF)
"\u{1F600}"    // "😀" — ES6 curly syntax, any code point
```

#### Normalisation

The same visible character can be encoded in multiple ways (e.g. é as one code point or as e + combining accent).

```js
const a = "é";         // é — single code point
const b = "é";        // é — e + combining accent

a === b           // false — different encodings
a.normalize() === b.normalize() // true — normalised to NFC
```


### ✅ Bitwise Operators & Tricks

#### Bitwise operators

Operate on **32-bit integers** — operands are converted to 32-bit signed integers, the operation is applied bit by bit, then converted back to a JS number.

| Operator | Name | Example | Result |
|----------|------|---------|--------|
| `&` | AND | `5 & 3` | `1` |
| `\|` | OR | `5 \| 3` | `7` |
| `^` | XOR | `5 ^ 3` | `6` |
| `~` | NOT | `~5` | `-6` |
| `<<` | Left shift | `5 << 1` | `10` |
| `>>` | Right shift (signed) | `5 >> 1` | `2` |
| `>>>` | Right shift (unsigned) | `-1 >>> 0` | `4294967295` |

#### Practical tricks

**Fast integer truncation (instead of `Math.floor` for positive numbers):**

```js
~~3.7      // 3   — double NOT truncates to integer
3.7 | 0    // 3   — OR with 0 truncates
-3.7 | 0   // -3  — note: truncates toward zero, not floor
```

**Check even / odd:**

```js
5 & 1   // 1 — odd
4 & 1   // 0 — even
```

**Multiply / divide by powers of 2:**

```js
5 << 1   // 10  — multiply by 2
5 << 2   // 20  — multiply by 4
8 >> 1   // 4   — divide by 2
```

**Toggle a flag using XOR:**

```js
let flags = 0b0000;
const READ  = 0b0001;
const WRITE = 0b0010;

flags |= READ;         // set READ flag   → 0b0001
flags |= WRITE;        // set WRITE flag  → 0b0011
flags ^= READ;         // toggle READ off → 0b0010
!!(flags & WRITE)      // true — WRITE is set
```

**Fast `Math.abs` alternative:**

```js
// Standard
Math.abs(-5) // 5

// Bitwise (works for 32-bit integers only)
const n = -5;
const mask = n >> 31;      // all 1s if negative, all 0s if positive
(n ^ mask) - mask;         // 5
```

> Bitwise tricks trade readability for micro-optimisation. Use only where performance matters and comment clearly.


### ✅ Proxy & Reflect

#### Proxy

A **Proxy** wraps an object and intercepts operations on it — reads, writes, deletes, function calls, etc.

```js
const handler = {
  get(target, key) {
    console.log(`Getting ${key}`);
    return key in target ? target[key] : `Property "${key}" not found`;
  },
  set(target, key, value) {
    if (typeof value !== "number") throw new TypeError("Only numbers allowed");
    target[key] = value;
    return true; // must return true to indicate success
  },
};

const obj = new Proxy({}, handler);

obj.x = 42;         // sets fine
obj.x;              // logs "Getting x", returns 42
obj.y;              // logs "Getting y", returns 'Property "y" not found'
obj.z = "hello";    // throws TypeError
```

#### Common traps

| Trap | Intercepts |
|------|-----------|
| `get` | Property read (`obj.x`) |
| `set` | Property write (`obj.x = 1`) |
| `has` | `in` operator (`"x" in obj`) |
| `deleteProperty` | `delete obj.x` |
| `apply` | Function call (`fn()`) |
| `construct` | `new` operator |

#### Reflect

**Reflect** provides methods that mirror Proxy traps — the default behaviour for each trap. Used inside Proxy handlers to forward the operation to the original object.

```js
const handler = {
  get(target, key, receiver) {
    console.log(`Getting ${key}`);
    return Reflect.get(target, key, receiver); // default behaviour
  },
};

const proxy = new Proxy({ name: "Atik" }, handler);
proxy.name; // logs "Getting name", returns "Atik"
```

#### Use cases

```js
// Validation proxy
const validated = new Proxy({}, {
  set(target, key, value) {
    if (key === "age" && (typeof value !== "number" || value < 0)) {
      throw new RangeError("age must be a non-negative number");
    }
    return Reflect.set(target, key, value);
  },
});

validated.age = 25;   // ok
validated.age = -1;   // throws RangeError
```

```js
// Default values proxy
function withDefaults(target, defaults) {
  return new Proxy(target, {
    get(obj, key) {
      return key in obj ? obj[key] : defaults[key];
    },
  });
}

const config = withDefaults({}, { theme: "dark", lang: "en" });
config.theme; // "dark" — from defaults
```


### ✅ Tagged Template Literals

A **tagged template** is a function call where the function receives the template literal's parts — lets you process the string and interpolated values however you want.

```js
function tag(strings, ...values) {
  console.log(strings); // ["Hello, ", "! You are ", " years old."]
  console.log(values);  // ["Atik", 30]
}

const name = "Atik";
const age = 30;

tag`Hello, ${name}! You are ${age} years old.`;
```

- `strings` — array of raw string parts between interpolations
- `...values` — the evaluated interpolated expressions

#### Building a custom result

```js
function highlight(strings, ...values) {
  return strings.reduce((result, str, i) => {
    return result + str + (values[i] !== undefined ? `<b>${values[i]}</b>` : "");
  }, "");
}

const user = "Atik";
const score = 95;

highlight`User ${user} scored ${score} points.`;
// "User <b>Atik</b> scored <b>95</b> points."
```

#### `String.raw`

A built-in tag that returns the raw string without processing escape sequences.

```js
String.raw`Hello\nWorld`  // "Hello\\nWorld" — backslash-n is not a newline
`Hello\nWorld`            // "Hello\nWorld"  — newline processed
```

#### Use cases

- SQL query builders (safe parameterised queries)
- HTML/CSS-in-JS (e.g. `styled-components`)
- Internationalisation (i18n)
- Input sanitisation


### ✅ Miscellaneous Operators

#### `void` operator

Evaluates an expression and returns `undefined`. Mainly used to explicitly return `undefined` or prevent navigation in anchor tags.

```js
void 0          // undefined
void "anything" // undefined
void (2 + 2)    // undefined
```

```html
<!-- Prevent navigation while running JS -->
<a href="javascript:void(0)" onclick="doSomething()">Click</a>
```

#### Comma operator

Evaluates multiple expressions left to right and returns the **last** value.

```js
const x = (1, 2, 3);   // x = 3
let i = (0, 10);        // i = 10
```

Rarely used directly — mostly appears in `for` loop initialisers.

```js
for (let i = 0, j = 10; i < j; i++, j--) {
  console.log(i, j);
}
```

#### `typeof` quirks

```js
typeof undefined    // "undefined"
typeof null         // "object"   ← historical bug
typeof 42           // "number"
typeof "str"        // "string"
typeof true         // "boolean"
typeof Symbol()     // "symbol"
typeof 42n          // "bigint"
typeof {}           // "object"
typeof []           // "object"   ← arrays are objects
typeof function(){} // "function"
typeof undeclared   // "undefined" ← no ReferenceError (safe to check)
```

#### Nullish coalescing `??` vs OR `||`

```js
// || returns right side for ANY falsy value
0 || "default"     // "default" — 0 is falsy
"" || "default"    // "default" — "" is falsy

// ?? returns right side only for null or undefined
0 ?? "default"     // 0   — 0 is not null/undefined
"" ?? "default"    // ""  — "" is not null/undefined
null ?? "default"  // "default"
undefined ?? "default" // "default"
```

#### Optional chaining `?.`

Short-circuits to `undefined` if the left side is `null` or `undefined` — avoids nested null checks.

```js
const user = null;

user.profile.name           // TypeError: Cannot read properties of null
user?.profile?.name         // undefined — safe

user?.greet()               // undefined — safe method call
arr?.[0]                    // safe array access
```
