## 15. Type Coercion & Equality

### ✅ Implicit vs Explicit Coercion

**Type coercion** is the automatic or manual conversion of a value from one type to another.

#### Explicit Coercion

You intentionally convert a type using built-in functions.

```js
Number("42");        // 42
String(100);         // "100"
Boolean(0);          // false
parseInt("3.14");    // 3
parseFloat("3.14");  // 3.14
```

#### Implicit Coercion

JavaScript automatically converts types behind the scenes.

```js
"5" - 2       // 3   — string "5" coerced to number
"5" + 2       // "52" — number 2 coerced to string
true + 1      // 2   — true coerced to 1
false + 1     // 1   — false coerced to 0
null + 1      // 1   — null coerced to 0
undefined + 1 // NaN — undefined coerced to NaN
```

Implicit coercion is triggered by operators (`+`, `-`, `*`, `==`, etc.) and conditionals (`if`, `while`, `? :`).


### ✅ `==` vs `===`

#### Strict Equality `===`

Compares **value AND type** — no coercion.

```js
1 === 1        // true
1 === "1"      // false — different types
null === null  // true
null === undefined // false
```

#### Abstract Equality `==`

Compares **value only** — may coerce types first.

```js
1 == "1"       // true  — "1" coerced to 1
0 == false     // true  — false coerced to 0
null == undefined // true  — special rule
null == 0      // false — null only equals undefined
NaN == NaN     // false — NaN is never equal to anything
```

#### Abstract Equality Rules (simplified)

| Comparison | Coercion applied |
|---|---|
| `number == string` | string → number |
| `boolean == anything` | boolean → number first |
| `object == primitive` | object → primitive (via `valueOf`/`toString`) |
| `null == undefined` | always `true`, no coercion |
| `null == anything else` | always `false` |
| `NaN == NaN` | always `false` |

**Rule of thumb:** Always use `===`. Only use `==` if you intentionally want `null == undefined` to be `true`.


### ✅ Truthy & Falsy

Every value in JavaScript is either **truthy** or **falsy** when used in a boolean context (`if`, `while`, `! `, `? :`).

#### The 6 Falsy Values

```js
false
0          // also: -0, 0n (BigInt zero)
""         // empty string ('' or ``)
null
undefined
NaN
```

Everything else is **truthy** — including:

```js
"0"        // truthy — non-empty string
"false"    // truthy — non-empty string
[]         // truthy — empty array
{}         // truthy — empty object
-1         // truthy — any non-zero number
```

#### Common pattern

```js
// Check if a value exists and is non-empty
if (value) { ... }

// Check for null or undefined specifically
if (value == null) { ... }   // catches both null and undefined
if (value != null) { ... }   // excludes both null and undefined
```


### ✅ Coercion Rules

#### To Number

```js
Number("")          // 0
Number("  ")        // 0
Number("42")        // 42
Number("42abc")     // NaN
Number(true)        // 1
Number(false)       // 0
Number(null)        // 0
Number(undefined)   // NaN
Number([])          // 0   — [] → "" → 0
Number([3])         // 3   — [3] → "3" → 3
Number([1,2])       // NaN — [1,2] → "1,2" → NaN
Number({})          // NaN
```

#### To String

```js
String(123)         // "123"
String(true)        // "true"
String(false)       // "false"
String(null)        // "null"
String(undefined)   // "undefined"
String([1, 2, 3])   // "1,2,3"
String({})          // "[object Object]"
```

#### To Boolean

```js
Boolean(0)          // false
Boolean("")         // false
Boolean(null)       // false
Boolean(undefined)  // false
Boolean(NaN)        // false
Boolean("0")        // true
Boolean([])         // true
Boolean({})         // true
```


### ✅ The `+` Operator

`+` is overloaded — it does **addition** for numbers and **concatenation** for strings. The rule: if either operand is a string, the other is converted to a string.

```js
1 + 2          // 3    — both numbers
1 + "2"        // "12" — number coerced to string
"1" + 2        // "12" — number coerced to string
"1" + 2 + 3    // "123" — left to right: "12" then "123"
1 + 2 + "3"    // "33" — left to right: 3 then "33"
```

#### `+` with other types

```js
true + true    // 2   — booleans coerced to numbers
true + false   // 1
[] + []        // ""  — both arrays → "" + "" = ""
[] + {}        // "[object Object]"
{} + []        // 0   — {} parsed as block, +[] = 0 (in statements)
```

#### Other arithmetic operators (`-`, `*`, `/`, `%`)

These always convert operands to **numbers** — no string concatenation.

```js
"10" - 4       // 6
"10" * "2"     // 20
"10" / "2"     // 5
true * 3       // 3
null * 5       // 0
undefined * 5  // NaN
```


### ✅ Object to Primitive

When an object is used in a context that expects a primitive, JavaScript calls internal methods to convert it. The process depends on the **hint**: `"number"`, `"string"`, or `"default"`.

#### Conversion order

1. If `Symbol.toPrimitive` exists — call it with the hint.
2. If hint is `"string"` — try `toString()` then `valueOf()`.
3. If hint is `"number"` or `"default"` — try `valueOf()` then `toString()`.

#### Default `valueOf` and `toString`

```js
const obj = { val: 42 };

obj.valueOf()   // { val: 42 } — returns the object itself by default
obj.toString()  // "[object Object]"
```

Since `valueOf()` returns an object (not a primitive), JS falls back to `toString()`.

```js
const obj = {};
obj + 1        // "[object Object]1" — {} → "[object Object]" → concat
```

#### Custom `valueOf`

```js
const money = {
  amount: 100,
  valueOf() { return this.amount; },
};

money + 50     // 150 — valueOf returns 100
`${money}`     // "100" — template literal calls toString, falls back to valueOf
```

#### `Symbol.toPrimitive`

Full control — called with a hint string.

```js
const temp = {
  celsius: 22,
  [Symbol.toPrimitive](hint) {
    if (hint === "number") return this.celsius;
    if (hint === "string") return `${this.celsius}°C`;
    return this.celsius; // default
  },
};

+temp          // 22         — number hint
`${temp}`      // "22°C"     — string hint
temp + 0       // 22         — default hint
```


### ✅ Common Gotchas

#### `null` and `undefined` behaviour

```js
null == undefined   // true  — only pair that is loosely equal
null === undefined  // false — different types
null + 1            // 1     — null coerced to 0
undefined + 1       // NaN   — undefined coerced to NaN
typeof null         // "object" — historical bug, not a real object
```

#### `NaN` is never equal to itself

```js
NaN == NaN    // false
NaN === NaN   // false

// Correct way to check
Number.isNaN(NaN)   // true
Number.isNaN("NaN") // false — does NOT coerce (unlike the global isNaN)
isNaN("NaN")        // true  — coerces first, then checks
```

#### Comparison with objects

```js
[] == false    // true  — [] → "" → 0, false → 0
[] == ![]      // true  — ![] is false → 0, [] → 0
{} == false    // false — {} → "[object Object]" → NaN, false → 0
```

#### `+` unary converts to number

```js
+"42"     // 42
+true     // 1
+false    // 0
+null     // 0
+""       // 0
+[]       // 0
+{}       // NaN
+undefined // NaN
```
