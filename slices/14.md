## 14. Memory & Performance

### ✅ Memory Life Cycle

Every value in JavaScript goes through three stages:

1. **Allocate** — memory is reserved when you declare a variable, create an object, or call a function.
2. **Use** — the value is read or written.
3. **Release** — memory is freed when the value is no longer reachable.

```js
// Allocation
const user = { name: "Atik" }; // object allocated on heap
const name = "Atik";           // string allocated on heap, reference on stack

// Use
console.log(user.name);

// Release — happens automatically when 'user' goes out of scope
// or when nothing holds a reference to it
```

JavaScript handles release automatically via the **Garbage Collector** — you don't free memory manually.

### ✅ Garbage Collection

The **Garbage Collector (GC)** automatically frees memory that is no longer reachable.

#### Mark and Sweep (modern algorithm)

1. GC starts from **roots** (global variables, call stack).
2. It **marks** everything reachable from roots.
3. Everything **not marked** is swept (freed).

```js
let user = { name: "Atik" }; // reachable — referenced by 'user'
user = null;                  // original object is now unreachable → GC frees it
```

#### Reachability

An object is kept alive as long as something holds a reference to it.

```js
let a = { val: 1 };
let b = a;        // two references to the same object

a = null;         // object still reachable via 'b'
b = null;         // now unreachable → eligible for GC
```

#### When GC runs

- You cannot control when GC runs — the engine decides.
- GC pauses JS execution briefly (stop-the-world) — frequent large allocations cause noticeable pauses.
- V8 uses **incremental** and **concurrent** GC to minimise these pauses.

### ✅ Common Memory Leaks

A **memory leak** happens when memory is allocated but never released — the GC can't free it because something still holds a reference, even unintentionally.

#### 1. Accidental global variables

```js
function leak() {
  data = "I am global"; // missing 'let/const/var' — attaches to window
}
```

**Fix:** Always use `let`, `const`, or `var`. Use `"use strict"` to catch this.

#### 2. Forgotten timers

```js
const timer = setInterval(() => {
  console.log(heavyData); // heavyData stays in memory as long as timer runs
}, 1000);

// Fix: clear the timer when done
clearInterval(timer);
```

#### 3. Forgotten event listeners

```js
const button = document.getElementById("btn");

button.addEventListener("click", handleClick);

// Fix: remove the listener when no longer needed
button.removeEventListener("click", handleClick);
```

#### 4. Detached DOM nodes

```js
let detached;

function detach() {
  const el = document.getElementById("box");
  detached = el;           // JS still holds a reference
  el.remove();             // removed from DOM but NOT garbage collected
}

// Fix: release the reference when no longer needed
detached = null;
```

#### 5. Closures holding large data

```js
function outer() {
  const largeArray = new Array(1_000_000).fill("data");

  return function inner() {
    console.log(largeArray[0]); // largeArray stays alive as long as inner exists
  };
}

const fn = outer(); // largeArray is never freed while fn exists
```

**Fix:** Only capture what you need, or set `fn = null` when done.

### ✅ `WeakMap`, `WeakSet`, `WeakRef`

These are GC-friendly data structures — they hold **weak references** that don't prevent garbage collection.

#### `WeakMap`

- Keys must be **objects** (not primitives).
- If the key object is garbage collected, the entry is automatically removed.
- Not iterable — no `.keys()`, `.values()`, `.forEach()`.

```js
const cache = new WeakMap();

let user = { name: "Atik" };
cache.set(user, { score: 100 });

console.log(cache.get(user)); // { score: 100 }

user = null; // user object is GC'd → cache entry is automatically removed
```

**Use case:** Attaching private data to objects without preventing GC.

#### `WeakSet`

- Stores **objects** only.
- If the object is GC'd, it's automatically removed from the set.
- Not iterable.

```js
const seen = new WeakSet();

let obj = { id: 1 };
seen.add(obj);

console.log(seen.has(obj)); // true

obj = null; // obj is GC'd → removed from WeakSet automatically
```

**Use case:** Tracking objects without keeping them alive (e.g. visited nodes).

#### `WeakRef`

- Holds a **weak reference** to an object — doesn't prevent GC.
- Use `.deref()` to access the object — returns `undefined` if already GC'd.

```js
let obj = { name: "Atik" };
const ref = new WeakRef(obj);

console.log(ref.deref()?.name); // "Atik"

obj = null; // may be GC'd

// later...
console.log(ref.deref()); // undefined (if GC'd)
```

#### Quick comparison

| | `WeakMap` | `WeakSet` | `WeakRef` |
|---|---|---|---|
| Stores | key-value pairs | values | single reference |
| Key/value type | object keys | objects only | any object |
| Prevents GC | ❌ No | ❌ No | ❌ No |
| Iterable | ❌ No | ❌ No | — |

### ✅ Lazy Loading

Defer loading resources until they are actually needed — reduces initial load time and memory usage.

#### Images — `loading="lazy"`

```html
<img src="photo.jpg" loading="lazy" alt="photo" />
```

The browser only loads the image when it's about to enter the viewport.

#### Dynamic imports

```js
// Eager — loaded upfront even if never used
import { heavyModule } from "./heavy.js";

// Lazy — loaded only when needed
async function loadChart() {
  const { renderChart } = await import("./chart.js");
  renderChart();
}
```

#### Intersection Observer for lazy loading

```js
const observer = new IntersectionObserver((entries) => {
  entries.forEach((entry) => {
    if (entry.isIntersecting) {
      const img = entry.target;
      img.src = img.dataset.src; // load image only when visible
      observer.unobserve(img);
    }
  });
});

document.querySelectorAll("img[data-src]").forEach((img) => {
  observer.observe(img);
});
```

### ✅ Optimize Loops

Small changes in loops can have a big impact when iterating over large datasets.

#### Cache array length

```js
const arr = new Array(100_000).fill(1);

// Slow — .length is re-evaluated every iteration
for (let i = 0; i < arr.length; i++) { }

// Fast — length cached
for (let i = 0, len = arr.length; i < len; i++) { }
```

#### Avoid DOM access inside loops

```js
// Slow — reads from DOM on every iteration
for (let i = 0; i < 1000; i++) {
  document.getElementById("box").style.left = i + "px";
}

// Fast — cache the DOM reference
const box = document.getElementById("box");
for (let i = 0; i < 1000; i++) {
  box.style.left = i + "px";
}
```

#### Avoid creating objects inside loops

```js
// Slow — new object created on every iteration
for (let i = 0; i < 10000; i++) {
  process({ index: i }); // new object each time → GC pressure
}

// Fast — reuse a single object
const item = { index: 0 };
for (let i = 0; i < 10000; i++) {
  item.index = i;
  process(item);
}
```

#### Use the right method

```js
// for loop is fastest for raw iteration
// forEach is clean but slightly slower
// for...of is readable, good middle ground
// map/filter/reduce add overhead — avoid chaining them on large arrays
```

### ✅ Debounce & Throttle (recap)

Already covered in depth in [slice 04](./04.md). Quick reminder:

| | Debounce | Throttle |
|---|---|---|
| Fires | After user **stops** triggering | At most once per **interval** |
| Use case | Search input, resize | Scroll, mousemove |

```js
// Debounce — wait until user stops typing
input.addEventListener("input", debounce(search, 300));

// Throttle — fire at most once every 200ms while scrolling
window.addEventListener("scroll", throttle(updateUI, 200));
```

### ✅ `performance` API

The built-in `performance` API lets you measure how long things take with high precision.

#### `performance.now()`

Returns a high-resolution timestamp in milliseconds (sub-millisecond precision).

```js
const start = performance.now();

// some work
for (let i = 0; i < 1_000_000; i++) {}

const end = performance.now();
console.log(`Took ${end - start}ms`);
```

#### `performance.mark()` & `performance.measure()`

Mark named points and measure the gap between them.

```js
performance.mark("start-fetch");

await fetch("/api/data");

performance.mark("end-fetch");

performance.measure("fetch-duration", "start-fetch", "end-fetch");

const [measure] = performance.getEntriesByName("fetch-duration");
console.log(`Fetch took ${measure.duration}ms`);
```

#### `performance.getEntries()`

Returns all recorded performance entries.

```js
performance.getEntries();        // all entries
performance.getEntriesByType("measure"); // only measures
performance.clearMarks();        // clear all marks
performance.clearMeasures();     // clear all measures
```
