## 12. Iterables & Generators

### ✅ Iterable vs Iterator Protocol

#### Iterator Protocol

An **iterator** is an object that implements the iterator protocol.
It has a `next()` method that returns an object with:

- `value` — the next value in the sequence
- `done` — a boolean indicating if the iteration is complete

```js
// custom iterator that counts from 1 to 3
function createRangeIterator(start, end) {
  let current = start;

  return {
    next: function () {
      if (current <= end) {
        return { value: current++, done: false };
      } else {
        return { value: undefined, done: true };
      }
    },
  };
}

const rangeIterator = createRangeIterator(1, 3);

console.log(rangeIterator.next()); // { value: 1, done: false }
console.log(rangeIterator.next()); // { value: 2, done: false }
console.log(rangeIterator.next()); // { value: 3, done: false }
console.log(rangeIterator.next()); // { value: undefined, done: true }
```

#### Iterable Protocol

An **iterable** is an object that implements the iterable protocol.
It must have a `[Symbol.iterator]()` method that **returns an iterator**.

Iterables are used in:
- `for...of` loops
- Spread syntax (`...`)
- Destructuring assignments

```js
// custom iterable
function createRangeIterable(start, end) {
  return {
    [Symbol.iterator]: function () {
      let current = start;

      return {
        next: function () {
          if (current <= end) {
            return { value: current++, done: false };
          } else {
            return { value: undefined, done: true };
          }
        },
      };
    },
  };
}

const rangeIterable = createRangeIterable(1, 3);

for (let num of rangeIterable) {
  console.log(num); // 1, 2, 3
}
```

#### Key Distinction

| | Iterator | Iterable |
|---|---|---|
| Has | `next()` method | `[Symbol.iterator]()` method |
| Role | Does the iterating | Produces an iterator |
| Works with `for...of` | No (directly) | Yes |

An object can be **both** — if `[Symbol.iterator]()` returns `this` and also has `next()`.


### ✅ `Symbol.iterator`

`Symbol.iterator` is a built-in Symbol that marks an object as iterable.
Any object with this method works with `for...of`, spread, and destructuring.

```js
// built-in iterable example (array)
const array = [1, 2, 3];

for (const value of array) {
  console.log(value); // 1, 2, 3
}

const copy = [...array];   // [1, 2, 3]
const [a, b, c] = array;  // 1 2 3
```

You can also call `Symbol.iterator` manually to get the iterator:

```js
const arr = [10, 20, 30];
const iter = arr[Symbol.iterator]();

console.log(iter.next()); // { value: 10, done: false }
console.log(iter.next()); // { value: 20, done: false }
console.log(iter.next()); // { value: 30, done: false }
console.log(iter.next()); // { value: undefined, done: true }
```

Plain objects `{}` are **not** iterable by default — you need to add `[Symbol.iterator]` manually (like `createRangeIterable` above).


### ✅ Built-in Iterables

These are iterable out of the box:

| Type | Example |
|------|---------|
| `Array` | `[1, 2, 3]` |
| `String` | `"hello"` |
| `Map` | `new Map()` |
| `Set` | `new Set()` |
| `arguments` object | inside functions |
| `NodeList` | `document.querySelectorAll(...)` |
| Generator objects | result of calling a `function*` |

```js
// String iterates over characters
for (const char of "hello") {
  console.log(char); // h e l l o
}

// Map iterates over [key, value] pairs
const map = new Map([["a", 1], ["b", 2]]);
for (const [key, val] of map) {
  console.log(key, val); // a 1 / b 2
}

// Set iterates over unique values
const set = new Set([1, 2, 2, 3]);
for (const val of set) {
  console.log(val); // 1 2 3
}
```


### ✅ `for...of` with Iterables

`for...of` calls `[Symbol.iterator]()` internally and repeatedly calls `next()` until `done: true`.

```js
// These two are equivalent:

for (const num of rangeIterable) {
  console.log(num);
}

// under the hood:
const iter = rangeIterable[Symbol.iterator]();
let result = iter.next();
while (!result.done) {
  console.log(result.value);
  result = iter.next();
}
```

#### `for...of` vs `for...in`

| | `for...of` | `for...in` |
|---|---|---|
| Iterates over | **Values** (via iterator protocol) | **Keys** (enumerable properties) |
| Works with | Arrays, Strings, Maps, Sets, custom iterables | All objects |
| Use for arrays | Yes | Avoid — includes inherited properties |


### ✅ `function*` & `yield`

A **generator** is a special function declared with `function*`.
It returns a **generator object**, which is both an iterator and an iterable.

Key features:
- Uses `yield` to **pause** execution and produce a value
- Resumes with `.next()`
- Can `return` a final value — sets `done: true`

```js
function* numberGenerator() {
  yield 1;
  yield 2;
  yield 3;
  return 4;
}

const gen = numberGenerator();

console.log(gen.next()); // { value: 1, done: false }
console.log(gen.next()); // { value: 2, done: false }
console.log(gen.next()); // { value: 3, done: false }
console.log(gen.next()); // { value: 4, done: true }
```

**Calling the function doesn't run it** — it returns a generator object. Execution starts only on the first `.next()`.

#### `yield` vs `return`

```js
function* pex() {
  yield "Hello"; // pauses, done: false
  return "World"; // ends, done: true
}

const p = pex();

console.log(p.next()); // { value: 'Hello', done: false }
console.log(p.next()); // { value: 'World', done: true }
```

- `yield` — pauses and hands back a value, can be resumed
- `return` — ends the generator, `done` becomes `true`

#### Generator is iterable

Generator objects implement both protocols — they work in `for...of` and spread.

```js
function* nums() {
  yield 1;
  yield 2;
  yield 3;
}

console.log([...nums()]); // [1, 2, 3]

for (const n of nums()) {
  console.log(n); // 1 2 3
}
```

Note: `return` values are **excluded** from `for...of` and spread — only `yield` values are consumed.

#### Passing values into `yield`

You can send a value **back into** the generator via `next(value)`. That value becomes the result of the `yield` expression inside the generator.

```js
function* adder() {
  const a = yield "Enter first number";
  const b = yield "Enter second number";
  return a + b;
}

const g = adder();

g.next();      // starts execution, pauses at first yield  → { value: 'Enter first number', done: false }
g.next(10);    // sends 10 as 'a', pauses at second yield → { value: 'Enter second number', done: false }
g.next(20);    // sends 20 as 'b', hits return            → { value: 30, done: true }
```

- The first `next()` call starts the generator — any value passed here is ignored.
- Each subsequent `next(value)` resumes execution and the value becomes what `yield` evaluated to.


### ✅ `yield*` — Delegating Generators

`yield*` delegates to another iterable (including another generator), yielding each of its values one by one.

```js
function* gen2() {
  yield 4;
  yield 5;
}

function* gen() {
  yield 1;
  yield* [2, 3];   // delegates to an array
  yield* gen2();   // delegates to another generator
  return 6;
}
```

**Consuming with `for...of`:**

```js
const g1 = gen();
const result1 = [];

for (let val of g1) {
  result1.push(val);
}

console.log(result1); // [1, 2, 3, 4, 5]
// return value (6) is excluded from for...of
```

**Consuming with `while` + `.next()`:**

```js
const g2 = gen();
const result2 = [];

let next = g2.next();
while (!next.done) {
  result2.push(next.value);
  next = g2.next();
}

console.log(result2); // [1, 2, 3, 4, 5]
```

Both approaches produce the same result — `for...of` is cleaner; the `while` loop gives you more control.

#### `yield*` returns the delegated generator's return value

```js
function* inner() {
  yield 1;
  return "done";
}

function* outer() {
  const result = yield* inner(); // result = "done"
  console.log(result);           // "done"
}

[...outer()]; // logs "done"
```


### ✅ Generator Methods: `return()` & `throw()`

#### `return(value)`

Forces the generator to finish immediately.

```js
function* gen() {
  yield 1;
  yield 2;
  yield 3;
}

const g = gen();
console.log(g.next());          // { value: 1, done: false }
console.log(g.return("stop")); // { value: 'stop', done: true }
console.log(g.next());          // { value: undefined, done: true }
```

#### `throw(error)`

Injects an error at the point the generator is paused. The generator can catch it with `try/catch`.

```js
function* safe() {
  try {
    yield 1;
    yield 2;
  } catch (err) {
    console.log("Caught:", err.message);
    yield "recovered";
  }
}

const g = safe();
console.log(g.next());                // { value: 1, done: false }
console.log(g.throw(new Error("!"))); // Caught: !  →  { value: 'recovered', done: false }
console.log(g.next());                // { value: undefined, done: true }
```


### ✅ Infinite Sequences & Lazy Evaluation

Generators compute values **on demand** — nothing is pre-computed until you call `.next()`.

#### Infinite ID generator

```js
function* idGenerator() {
  let id = 1;
  while (true) {
    yield id++;
  }
}

const getId = idGenerator();
console.log(getId.next().value); // 1
console.log(getId.next().value); // 2
console.log(getId.next().value); // 3
// safe — only computes when asked
```

#### Infinite Fibonacci

```js
function* fibonacci() {
  let [a, b] = [0, 1];
  while (true) {
    yield a;
    [a, b] = [b, a + b];
  }
}

const fib = fibonacci();
for (let i = 0; i < 8; i++) {
  process.stdout.write(fib.next().value + " ");
}
// 0 1 1 2 3 5 8 13
```

#### Lazy pipeline (no large arrays built in memory)

```js
function* range(start, end) {
  for (let i = start; i <= end; i++) yield i;
}

function* map(iter, fn) {
  for (const val of iter) yield fn(val);
}

function* filter(iter, pred) {
  for (const val of iter) {
    if (pred(val)) yield val;
  }
}

const result = filter(
  map(range(1, 1_000_000), (x) => x * 2),
  (x) => x % 6 === 0
);

console.log(result.next().value); // 6 — only computes what's needed
```


### ✅ Async Generators & `for await...of`

Async generators combine `async/await` with generators — useful for streaming async data page by page.

```js
async function* asyncRange(start, end) {
  for (let i = start; i <= end; i++) {
    await new Promise((res) => setTimeout(res, 100)); // async delay
    yield i;
  }
}

(async () => {
  for await (const num of asyncRange(1, 5)) {
    console.log(num); // 1 2 3 4 5 (with delay)
  }
})();
```

#### Paginated API example

```js
async function* fetchPages(url) {
  let page = 1;
  while (true) {
    const res = await fetch(`${url}?page=${page}`);
    const data = await res.json();
    if (data.items.length === 0) return;
    yield data.items;
    page++;
  }
}

(async () => {
  for await (const items of fetchPages("https://api.example.com/posts")) {
    console.log("Page items:", items);
  }
})();
```


### ✅ Use Cases

| Use Case | How generators help |
|---|---|
| Infinite sequences | `while(true)` with `yield` — compute on demand |
| Lazy evaluation | No large arrays built — process only what's needed |
| Pagination / streaming | `async function*` — fetch next page only when needed |
| Custom iterables | Implement `[Symbol.iterator]` with a generator |
| State machines | `yield` naturally pauses at each state transition |
| Unique ID generation | Stateful counter without globals |

#### State machine example

```js
function* trafficLight() {
  while (true) {
    yield "green";
    yield "yellow";
    yield "red";
  }
}

const light = trafficLight();
console.log(light.next().value); // green
console.log(light.next().value); // yellow
console.log(light.next().value); // red
console.log(light.next().value); // green (loops)
```
