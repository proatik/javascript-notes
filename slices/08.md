## 8. Error Handling

### ‚úÖ `try` / `catch` / `finally`

- Used to handle runtime errors gracefully without breaking execution.
- `finally` always executes, whether error happens or not.

```js
try {
  let x = JSON.parse("{ bad json }");
} catch (err) {
  console.error("Error:", err.message);
} finally {
  console.log("Cleanup always runs");
}
```

### ‚úÖ `throw`

- You can throw your own errors using `throw`.
- Any type can be thrown, but usually `Error` objects are preferred.

```js
function divide(a, b) {
  if (b === 0) {
    throw new Error("Division by zero!");
  }
  return a / b;
}
```

### ‚úÖ Custom Errors

- Create user-defined errors by extending `Error` class.

```js
class ValidationError extends Error {
  constructor(message) {
    super(message);
    this.name = "ValidationError";
  }
}

try {
  throw new ValidationError("Invalid input");
} catch (err) {
  console.error(err.name, err.message);
}
```

### ‚úÖ Error Types

- **`Error`** ‚Üí Base class.
- **`SyntaxError`** ‚Üí Invalid code syntax.
- **`ReferenceError`** ‚Üí Using a variable that isn‚Äôt defined.
- **`TypeError`** ‚Üí Operation on wrong type.
- **`RangeError`** ‚Üí Value out of range (e.g., invalid array length).
- **`EvalError`** ‚Üí Related to `eval()` (rare).
- **`URIError`** ‚Üí Incorrect usage of URI functions.

```js
try {
  null.f(); // TypeError
} catch (e) {
  console.log(e.name); // "TypeError"
}
```

### ‚úÖ Error Object Properties

Every error object has:

- `name` ‚Üí Type of error (`TypeError`, `SyntaxError`, etc.).
- `message` ‚Üí Error message string.
- `stack` ‚Üí Call stack trace (debugging info).

```js
try {
  throw new Error("Something went wrong");
} catch (e) {
  console.log(e.name); // "Error"
  console.log(e.message); // "Something went wrong"
  console.log(e.stack); // Stack trace
}
```
