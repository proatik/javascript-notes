## 9. Modules

### ✅ ES6 `import` / `export`

- Modules let you split code into reusable files.
- Each file is a separate **module** with its own scope.

```js
// math.js
export const add = (a, b) => a + b;

export const sub = (a, b) => a - b;
```

```js
// main.js
import { add, sub } from "./math.js";

console.log(add(3, 2)); // 5
console.log(sub(5, 3)); // 2
```

### ✅ Default vs Named Exports

- **Named exports:** You can export multiple values. Must use same names when importing.
- **Default export:** Only one per file. Can be imported with any name.

```js
// named exports
export const PI = 3.14;

export function area(r) {
  return PI * r * r;
}

// default export
export default function greet(name) {
  console.log(`Hello, ${name}`);
}
```

```js
// importing
import greet, { PI, area } from "./utils.js";

greet("Atik");

console.log(area(5));
```

### ✅ Module Scope

- Variables/functions inside a module are **scoped** to that file.
- They are **not global**, even if declared with `var`.
- Use exports to make them accessible outside.

```js
// inside module.js
let secret = 42;

export const reveal = () => secret;
```

### ✅ Dynamic Imports

- Import modules **on demand** at runtime.
- Returns a Promise that resolves to the module object.

```js
// dynamic import
button.addEventListener("click", async () => {
  const math = await import("./math.js");
  console.log(math.add(2, 3));
});
```

#### When to Use

- Lazy loading (load only when needed).
- Conditional imports (e.g., based on user action or environment).

### ✅ Module Execution

- Each module is evaluated **only once** (cached by the loader).
- Imports are **live bindings** — if the exported value changes, import reflects it.

```js
// counter.js
export let count = 0;

export function inc() {
  count++;
}
```

```js
// main.js
import { count, inc } from "./counter.js";

inc();

console.log(count); // 1 (live update)
```

### ✅ Importing Everything

- You can import all exports as a single object.

```js
import * as math from "./math.js";

console.log(math.add(2, 3));
console.log(math.sub(5, 1));
```

### ✅ Top-level `await`

- ES2022 allows using `await` directly inside modules.

```js
// data.js
const res = await fetch("/api/data");

export const data = await res.json();
```
