## **10. Browser APIs**

### **Web Storage**

### ✅ `localStorage`

- Persistent key–value storage (~5–10MB, browser-dependent).
- Data stays even after closing the browser.
- Stores **strings only**.

```js
localStorage.setItem("name", "Atik");
localStorage.getItem("name"); // "Atik"
localStorage.removeItem("name");
localStorage.clear();
```

### ✅ `sessionStorage`

- Temporary key–value storage for a single tab (~5-10MB, browser-dependent).
- Data is removed when **tab is closed**.
- Stores **strings only**.

```js
sessionStorage.setItem("step", "2");
sessionStorage.getItem("step"); // "2"
sessionStorage.removeItem("step");
sessionStorage.clear();
```

### ✅ Cookies

- Small key–value storage (~4KB).
- Sent automatically with every HTTP request.
- Can set expiration, path, domain, `secure`, `sameSite`, etc.

```js
document.cookie = "theme=dark; expires=Fri, 31 Dec 2028 23:59:59 GMT; path=/";
console.log(document.cookie);
```

- **HttpOnly**:

  - Cookie cannot be read or modified by JavaScript (`document.cookie`).
  - Protects against XSS stealing session tokens.
  - Still sent with requests to the server.

### ⚡ Storage Comparison

| Feature         | localStorage | sessionStorage   | Cookies            |
| --------------- | ------------ | ---------------- | ------------------ |
| Persistence     | Forever      | Until tab closes | Based on `expires` |
| Max Size        | ~5–10MB      | ~5–10MB          | ~4KB               |
| Sent to Server? | ❌           | ❌               | ✅                 |
| Use Case        | App data     | Per-tab state    | Auth, tracking     |

### ✅ IndexedDB

- Browser’s built-in **NoSQL database**.
- Stores large amounts of **structured data**.
- **Asynchronous** (Promise or event-based).
- Perfect for offline apps.

```js
const request = indexedDB.open("MyDB", 1);

request.onupgradeneeded = () => {
  const db = request.result;
  db.createObjectStore("users", { keyPath: "id" });
};

request.onsuccess = () => {
  const db = request.result;
  const tx = db.transaction("users", "readwrite");
  tx.objectStore("users").add({ id: 1, name: "Atik" });
};
```

### **Timers**

### ✅ `setTimeout`

```js
setTimeout(() => {
  console.log("Runs once after 1s");
}, 1000);
```

### ✅ `setInterval`

```js
const id = setInterval(() => {
  console.log("Repeats every 1s");
}, 1000);
```

### ✅ Clearing Timers

```js
clearTimeout(id);
clearInterval(id);
```

### **Clipboard & Geolocation**

### ✅ Clipboard API

#### Write text

```js
await navigator.clipboard.writeText("Hello!");
```

#### Read text

```js
const text = await navigator.clipboard.readText();
console.log(text);
```

### ✅ Geolocation API

```js
navigator.geolocation.getCurrentPosition((pos) => {
  console.log(pos.coords.latitude, pos.coords.longitude);
});
```

### **Web Workers**

- Run JavaScript in **background threads**.
- Prevent UI blocking.

### Worker file (`worker.js`)

```js
self.onmessage = (e) => {
  self.postMessage(e.data * 2);
};
```

### Main thread

```js
const worker = new Worker("worker.js");

worker.postMessage(10);

worker.onmessage = (e) => {
  console.log(e.data); // 20
};
```

### **Observers**

### ✅ Intersection Observer

Tracks when elements enter/leave the viewport.

```js
const io = new IntersectionObserver((entries) => {
  entries.forEach((e) => {
    if (e.isIntersecting) console.log("Visible:", e.target);
  });
});

io.observe(document.querySelector("#box"));
```

### ✅ Mutation Observer

Watches for DOM changes.
(Microtask queue → runs **before** rendering.)

```js
const target = document.getElementById("container");

const mo = new MutationObserver((mutations) => {
  console.log("DOM changed", mutations);
});

mo.observe(target, { childList: true, subtree: true });
```
